Pizza Views¶

Endpoint docs for pizza.views:

Views module

pizza.views.close(request, pizza_order_id)[source]¶

This view closes particular pizza order when pizza is delivered. After mark pizza as delivered this view redirects us to pizza:view

Parameters:
  • request – HttpRequest, accepted methods: GET
  • pizza_order_id – Integer, id of particular pizza order.
Returns:

view or errors: with status code 404 if order with provided id is not found with status code 405 if used another method (not GET)
Return type:

HttpResponse with redirect to pizza
pizza.views.create(request)[source]¶

This view renders a form for creating new PizzaOrder or accepts POST requests with the form to save its data. Or render errors.

PizzaOrder: is a base class that contains all order data, see field list below:

  • kind
  • size
  • delivery
  • extra
  • exclude
  • comment
  • delivered
  • date_created
  • date_delivered
Parameters:request – HttpRequest, accepted methods: GET, POST
Returns:HttpResponse with rendered view, or error code if used method other than GET or POST
pizza.views.get_stats_report(request, *args, **kwargs)[source]¶

This view shows various statistics about the created orders. It should be called after collection task is finished.

Parameters:
  • request – HttpRequest, accepted methods: GET
  • task_id – Integer, id of async task that is started by pizza:stats to collect statistics.
Returns:

with status code 404 if task with provided id is not found with status code 405 if used another method (not GET)
Return type:

HttpResponse with rendered view or errors
pizza.views.index(request)[source]¶

This view renders the main page, listing all orders. Every link in the list points to the create()

Parameters:request – HttpRequest, accepted methods: GET
Returns:HttpResponse with rendered view or error with status code 405 if used another method (not GET)
pizza.views.stats(request, *args, **kwargs)[source]¶

This view creates an async task to collect various statistics about the created orders. As a result returns an HttpResponse with task’s id.

Parameters:request – HttpRequest, allowed methods: GET
Returns:HttpResponse with task’s id or error if used method other than GET
pizza.views.view(request, pizza_order_id)[source]¶

This view shows us particular pizza order details (usually we can get here by link from orders page) Also select from related tables is performed (to show chosen values for this order)

  • pizza_order_id – Integer, id of particular pizza order.
Returns:
